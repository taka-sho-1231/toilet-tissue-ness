# Contributing

English version below

TL;DR
- PR は必ず dev 宛て
- Issue は気軽に作成OK（下書き歓迎）

## 開発フロー
1. 必要なら Issue を作成
2. dev から作業ブランチ作成（例: feature/..., fix/...）
3. 変更・ローカル確認
4. dev 宛てに小さめの PR

## Issue（任意で以下を参考に）
- 概要 / 期待と現状 / 再現手順（あれば）/ 環境

## English (EN)

TL;DR
- Always open PRs against dev
- Feel free to open issues (drafts welcome)

### Flow
1. (Optional) Open an issue for the idea/problem
2. Create a branch from dev (e.g., feature/..., fix/...)
3. Make changes and verify locally
4. Open a small PR targeting dev

### Issues (optional guide)
- Summary / Expected vs Current / Steps to Reproduce (if any) / Environment
